- **User Interface Prototyping**
    - Sketches, wireframes, mockups and prototypes are all representations of users interfaces used in user interface prototyping
        - We can refer to prototyping methods
    - Whats the difference?
        - A lot of these terms get used interchangeably
        - We're going to use a more proper granular definition. 
    
    - *Sketch Example*
    ![img]({{ site.url }}/assets/hc1.png)

    - *Sketches*
        - Hand drawn representation of user interface designs
            - Can be on paper, whiteboards, blackboards
        - *Zero barrier to entry*
            - Anyone can make a sketch, so non-technical customer can be involved
        - Cost in time in sketches is very low but also are low fidelity in quality (less overall detail)
        - High fidelity is closer to the final product and will get better overall feedback
        - We can mode interactivitiy with sketches and other low-fidelity prototyping methods


    - *Wireframes*
        - Accurate represent interface page layout and organization of ]
        - Analogous of blueprints, but left out of details but have a general scope of the UI will look like
        - Wireframe provide more concrete representations of designs made as sketches
        - Increased precision over sketches use software makes wireframes less accessible than sketches
        - Working interface should very closely resemble wireframe, certain details aside

    - *Wireframe Example*
    ![img]({{ site.url }}/assets/hc2.png)
    - *Mockups*
        - Are static but accurate representations of application interface
            - All graphics, typography and color are there
            - Mockups are not interactive, cannot click ont hem
        - Usually created with purpose-built mockup tools or graphics editors like Photoshop or Gimp
            - Can't sketch a mockup
            - Can make it HTML/CC but usually left for prototype
        - Mockups are even more time consuming than wireframes, but very high fidelity
    -*Mockup Example*
    ![img]({{ site.url }}/assets/hc4.png)
    - *Wireframe to Mockup Example*
    ![img]({{ site.url }}/assets/hc3.png)


    - *Prototypes*
        - Are an accurate and interactive representation of the application interface   
            - We can perform actions in a prototype and result should be the same in the full working application (eg. transition from one screen to next)
        - Difference between prototype and completed applications
            - Not connected to a backend and all of the related logic 
        - Most high fidelity and most time intensive to create
        - Can be used as same tools as completed applications
            - Saves development time when final step is to connect to back-end
        - Or prototypes can be completed with purpose-build tools that allow for simulated interactivity
        - UI is fully functional, which was several benefits
            - We can verify the UI with customer/user
            - We should be able make a user help guide using the prototype
            - Also can give it marketing and sales and they can start up on that process
