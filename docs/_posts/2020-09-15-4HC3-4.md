    - Short term and working memory
    - Long term and semantic memory

    - Arousal and vigilance
    - Fatigue and sleep deprivation
    - Drugs, smoking, and alcohol
    - Good website menus have 7 or so options
        -  Most adults can hold 7 +- 2 in their shorterm memory
            - makes a range of 5-8 menu items ideal
            - anymore than that and people will forget the first items they read by the time they read the last one
            - use submenus, but also limit 5-9 items
        - This applies to more than website menus

- How can universal deisgn benefit you?
    - ![img]({{ site.url }}/assets/hc14.png)
    
- Same Idea for UI Design!
    - Removing mouse-only interactions such as "hover to see info"
        - Helps users with vision or motor impairment, but now you can use the web app on your smartphone
    - High contrast information display
        - Helps users with vision impairment but also helps other users when they are standing in sunlight
    - Alt text on images?
        - Users who use screen readers, but also helps you when bandwith speed is too slow to send image
- Designing for all users can help **everyone**
- Should we always design for **everyone**?
    - NO
- Users and Design
    - Universal design and general philosophy is virtually always good to practice and to keep top of mind
    when designing
        - Especially for things like websites that provide, information intended to be universally variable
    - But designing for all possible users, all the time?
        - UI extremes like fighter pilot
    - Target users
        - Will our decisions about design be optimal if we're always considering every possible user equally?
        - What about interfaces
        - What about interfaces for specific type of workplace, like EMTs or fighters?
    - This is where user analysis becomes useful!
        - Understanding our users are better to influence our design
